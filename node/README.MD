 ### 原因：
 - js设计为了浏览器
 -  前端特点：不持久化，执行一遍就全部回收；js回收内存的时候，会暂停执行；
 -  新生代内存空间与老生代内存空间；
    - 新生代（from与to两个空间）
    1.简单说就是复制；临时变量
    - 老生代
    1.标记删除整理；
    - 新生代晋升老生代
    1.是否被回收过
    2.新生代to空间是否使用25%
- 为什么内存连续把他连续？
- 内存优化：尽量避免全局变量；匿名自执行函数将全局变为局部变量；尽量避免闭包引用；避免滥用缓存，（redis或者加锁，限制缓存的数据长度）；尽量避免大内存操作；
